| ------------------------------------------- | ---------------------------------------------------------------------------------- |
| Build process → generated fixture artifacts | Built JS/HTML/JSON assets are scanned after package and fixture commands complete. |
| Scanner constants → filesystem traversal    | Static scan roots determine which generated directories are inspected.             |
| Package script → shell execution            | Repo-local static command sequence invokes package and fixture build scripts.      |

## STRIDE Threat Register

| Threat ID  | Category               | Component                                  | Disposition | Mitigation Plan                                                                                                  |
| ---------- | ---------------------- | ------------------------------------------ | ----------- | ---------------------------------------------------------------------------------------------------------------- |
| T-03-03-01 | Tampering              | `scripts/check-hmr-leakage.mjs` scan roots | mitigate    | Use explicit generated output directories, not repo-wide search that skips ignored files.                        |
| T-03-03-02 | Information Disclosure | Generated artifacts                        | mitigate    | Denylist bridge/runtime strings that would expose dev-only HMR internals in shipped outputs.                     |
| T-03-03-03 | Elevation of Privilege | `package.json` command                     | mitigate    | Keep commands static and repo-local; scanner accepts no user-supplied shell fragments or paths.                  |
| T-03-03-04 | Denial of Service      | Fixture build command duration             | accept      | TEST-08 requires fixture builds; command is separate from default `pnpm test` to keep normal unit gates focused. |

</threat_model>
