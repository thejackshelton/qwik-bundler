## Trust Boundaries

| Vite dev server -> browser HMR channel | The smoke validates custom `qwik:hmr` transport through browser-observed `qHmr`.   |

## STRIDE Threat Register

| Threat ID  | Category          | Component                                   | Disposition | Mitigation Plan                                                                                                                   |
| ---------- | ----------------- | ------------------------------------------- | ----------- | --------------------------------------------------------------------------------------------------------------------------------- |
| T-04-01-01 | Tampering         | `fixtures/vite-csr/src/home.tsx`            | mitigate    | Read original contents first and restore them in a `finally` block; verify `git diff -- fixtures/vite-csr/src/home.tsx` is empty. |
| T-04-01-02 | Denial of Service | Browser smoke process/server lifecycle      | mitigate    | Close browser and Vite server in `finally`; use deterministic wait timeouts and fail with clear messages.                         |
| T-04-01-03 | Spoofing          | Browser event observation                   | mitigate    | Require both DOM marker update and a recorded browser `qHmr` event, not only a script-side log.                                   |
| T-04-01-04 | Tampering         | Source behavior changes after smoke failure | mitigate    | Compare relevant behavior against upstream Qwik Vite plugin before changing local source and keep fixes scoped to owner files.    |
