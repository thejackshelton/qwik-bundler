<threat_model>

## Trust Boundaries

| Boundary                       | Description                                                                          |
| ------------------------------ | ------------------------------------------------------------------------------------ |
| Vite dev server → browser HTML | Dev server injects a browser-executed module script into untrusted application HTML. |
| Vite HMR client → Qwik runtime | Browser bridge translates Vite custom events into document-level Qwik events.        |

## STRIDE Threat Register

| Threat ID  | Category               | Component                         | Disposition | Mitigation Plan                                                                                                                                                           |
| ---------- | ---------------------- | --------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-02-01-01 | Tampering              | `src/client/hmr-bridge.ts`        | mitigate    | Keep bridge source static, browser-only, and free of interpolated module graph data; tests assert the loaded virtual module contains the expected static bridge behavior. |
| T-02-01-02 | Information Disclosure | `src/vite.ts` / `src/vite/hmr.ts` | mitigate    | Gate HTML injection to Vite serve with `hmr !== false`; Phase 2 tests cover disabled non-injection and Phase 3 verifies build leakage.                                    |
| T-02-01-03 | Denial of Service      | `src/client/hmr-bridge.ts`        | mitigate    | Deduplicate repeated timestamps before dispatching and use a single cleared timeout for fallback reload.                                                                  |
| T-02-01-04 | Elevation of Privilege | Bridge virtual module             | accept      | Dev-server-only local HMR module has no auth or privilege boundary; no secrets or privileged APIs are introduced.                                                         |

</threat_model>
