<threat_model>

## Trust Boundaries

| Boundary                                | Description                                                                                            |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Vite module graph → browser HMR event   | File URLs from Vite module graph become browser-visible custom event payload data.                     |
| Vite helper → generic dev segment cache | Vite helper invokes a narrow invalidation API without exposing Vite internals to generic segment code. |

## STRIDE Threat Register

| Threat ID  | Category               | Component                           | Disposition | Mitigation Plan                                                                                              |
| ---------- | ---------------------- | ----------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------ |
| T-02-02-01 | Denial of Service      | `src/vite/hmr.ts` source collection | mitigate    | Filter payloads to source/importer JS/TS/MDX URLs and avoid broad module broadcasts.                         |
| T-02-02-02 | Tampering              | Segment invalidation path           | mitigate    | Use the Phase 1 `invalidateDevSegments(parent, 'client')` API and invalidate only returned graph module ids. |
| T-02-02-03 | Information Disclosure | `qwik:hmr` file payload             | mitigate    | Strip query/hash and avoid sending unrelated virtual/raw/CSS module URLs.                                    |
| T-02-02-04 | Repudiation            | HMR update handling                 | accept      | Dev-server HMR events are transient local development operations; tests provide behavioral auditability.     |

</threat_model>
